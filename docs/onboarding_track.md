# 新人 15 天培训路线

**作者：Manus AI**  
**版本：v0.1**  
**状态：试用期训练稿**

## 一、训练目标

本训练路线面向刚加入团队的新人，目标不是让其在 15 天内成为高级工程师，而是让他在 AI 与 SOP 的辅助下，快速具备参与日常交付的最低独立能力。换句话说，训练完成后，他至少应能完成基础环境配置、账号接入、仓库克隆、SSH 登录、简单部署协助、基础数据库连接、简单代码修改和任务回报。

整个训练设计强调“**从环境到协作、从协作到执行、从执行到交付**”。每一天都必须有明确产出，而不是只看教程。

## 二、考核原则

| 原则 | 说明 |
| --- | --- |
| 结果导向 | 每天必须有可验证输出 |
| 辅助允许 | 可以结合 AI 工具完成任务 |
| 真实任务优先 | 尽量把训练和真实项目小任务结合 |
| 可回溯 | 每项训练都应有截图、链接或提交记录 |
| 人工复核 | 阶段评分必须由负责人确认 |

## 三、15 天阶段划分

| 阶段 | 时间 | 目标 |
| --- | --- | --- |
| 阶段一 | Day 1-3 | 完成身份、工具、账号与本地环境准备 |
| 阶段二 | Day 4-7 | 学会代码仓库、SSH、部署平台和数据库基础操作 |
| 阶段三 | Day 8-11 | 能完成简单修改、部署配合与问题排查 |
| 阶段四 | Day 12-15 | 能独立完成小任务并提交交付记录 |

## 四、每日训练表

### Day 1：组织进入与工具安装

第一天的目标，是让新人具备进入团队工作空间的基础能力。重点不在开发，而在于把账号和沟通渠道打通。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 获得工作身份与基础工具 |
| 必做任务 | 登录企业邮箱、加入 Telegram 群、安装 WebStorm、安装 Claude Code、安装 Git |
| 输出物 | 工具安装截图、登录成功截图、设备与系统信息 |
| 评分重点 | 是否能独立完成安装与登录 |

### Day 2：代码与协作环境

第二天重点是熟悉代码协作入口，并能把本地开发环境跑起来。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 获取 GitHub 访问与本地仓库能力 |
| 必做任务 | 登录 GitHub、加入组织、配置 SSH Key、克隆示例仓库、成功拉取代码 |
| 输出物 | SSH 配置记录、仓库 clone 成功截图 |
| 评分重点 | 是否理解仓库、分支与提交基本概念 |

### Day 3：基础开发环境连通

第三天要让新人第一次把代码、IDE 和运行环境联系起来。重点是“能跑起来”，不是写复杂代码。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 能在本地打开项目并运行基础命令 |
| 必做任务 | 用 WebStorm 打开仓库、安装依赖、启动项目、读取 README、记录卡点 |
| 输出物 | 本地运行截图、问题清单 |
| 评分重点 | 是否能整理问题而不是只说“跑不起来” |

### Day 4：部署平台认知

第四天开始接触实际部署体系，理解项目不仅有代码，还有托管与域名。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 认识 Cloudflare / Vercel / Netlify 的角色 |
| 必做任务 | 登录部署平台、识别已有项目、查看环境变量位置、理解部署日志入口 |
| 输出物 | 各平台截图与说明 |
| 评分重点 | 是否知道哪个平台负责什么 |

### Day 5：SSH 与服务器基础

第五天进入远程开发与运维基础，这是后续独立处理任务的关键门槛。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 能连接到主开发服务器 |
| 必做任务 | 配置 SSH、登录服务器、查看目录、理解基础 Linux 命令 |
| 输出物 | SSH 登录截图、常用命令笔记 |
| 评分重点 | 是否能安全连接并描述所做操作 |

### Day 6：数据库连接基础

第六天重点是认识 Supabase 与数据库连接方式，而不是立即做复杂 SQL。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 能连接 Supabase 并查看基础数据结构 |
| 必做任务 | 登录 Supabase、识别项目、查看表结构、理解环境区分 |
| 输出物 | 表结构截图、术语说明 |
| 评分重点 | 是否理解生产库、测试库和权限差异 |

### Day 7：DNS 与域名基础

第七天让新人认识网站上线不只是“推代码”，还包括域名、DNS 和解析关系。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 理解域名解析与站点绑定 |
| 必做任务 | 查看 Cloudflare DNS、识别 A 记录 / CNAME、理解域名与站点关系 |
| 输出物 | DNS 记录截图、解释说明 |
| 评分重点 | 是否能区分域名、DNS、服务器和站点 |

### Day 8：简单代码修改

第八天开始从环境阶段过渡到执行阶段。任务应尽量简单，例如修改文案、样式、配置或非关键逻辑。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 完成一次小范围代码变更 |
| 必做任务 | 新建分支、修改代码、提交 commit、发起 PR 或补丁 |
| 输出物 | commit 链接、变更说明 |
| 评分重点 | 是否会说明改了什么、为什么这样改 |

### Day 9：日志与错误定位

第九天开始训练排查意识。重点不是一次解决所有问题，而是学会描述问题并定位入口。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 能读日志和定位错误入口 |
| 必做任务 | 查看构建日志、运行日志、找出一个报错来源、给出解释 |
| 输出物 | 报错截图、分析记录 |
| 评分重点 | 是否有结构化排查思路 |

### Day 10：订阅与资源记录

第十天让新人开始接触运营层工作，例如识别服务、订阅与续费责任。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 学会记录资产与订阅 |
| 必做任务 | 录入一个域名、一个服务器、一个订阅服务到资产台账 |
| 输出物 | 录入记录、字段说明 |
| 评分重点 | 是否能正确识别归属与到期时间 |

### Day 11：会议纪要转行动项

第十一天开始参与信息整理工作，让新人学会把沟通变成执行。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 能整理会议要点并抽取任务 |
| 必做任务 | 阅读一份纪要、提炼结论、列出行动项与负责人建议 |
| 输出物 | 纪要整理文档 |
| 评分重点 | 是否能区分结论、问题与待确认项 |

### Day 12：小任务独立执行

第十二天应开始进入“可交付”的状态。任务类型可以是更新文案、绑定域名、补配置、修简单 bug 等。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 独立完成一个小任务 |
| 必做任务 | 接单、执行、记录、回报 |
| 输出物 | 任务结果、提交链接、说明 |
| 评分重点 | 是否能独立推进而不是等待逐条指挥 |

### Day 13：交付表达训练

第十三天训练的是“完成了还要会汇报”。新人必须学会面向内部和客户写出清晰更新。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 输出一份结构清晰的任务汇报 |
| 必做任务 | 写任务背景、处理过程、结果、遗留问题 |
| 输出物 | 汇报文档 |
| 评分重点 | 是否表达清楚、简洁、可信 |

### Day 14：模拟综合任务

第十四天做一次综合演练，把前面的环境、协作、排查和汇报串起来。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 完成一次小型端到端任务 |
| 必做任务 | 拉代码、改配置、连接平台、提交结果、写总结 |
| 输出物 | 完整任务记录 |
| 评分重点 | 是否形成完整工作闭环 |

### Day 15：最终评估

第十五天不是再学新内容，而是确认其是否达到进入正式任务流的标准。

| 项目 | 要求 |
| --- | --- |
| 今日目标 | 完成试用期总结与最终评估 |
| 必做任务 | 自评、知识点回顾、负责人面评 |
| 输出物 | 试用总结、评分表 |
| 评分重点 | 是否具备独立承担基础任务的能力 |

## 五、评分维度

评分不应只看“做没做完”，而应综合看理解、独立性、规范性和沟通质量。

| 维度 | 权重 | 说明 |
| --- | --- | --- |
| 环境配置能力 | 20% | 是否能独立完成工具、账号、SSH、IDE 配置 |
| 协作能力 | 20% | 是否能正确使用 GitHub、任务流、汇报流程 |
| 执行能力 | 25% | 是否能完成简单任务 |
| 排查能力 | 15% | 是否能读日志、定位问题、提出假设 |
| 文档与沟通 | 10% | 是否能写清楚纪要与任务汇报 |
| 责任感与稳定性 | 10% | 是否按时反馈、遇阻不卡死 |

## 六、可交付标准

试用期结束时，合格的新人成员至少应满足以下要求：能够登录和使用核心平台；能够 clone 仓库并完成基础改动；能够连接主开发服务器和 Supabase；能够查看部署状态和 DNS 配置；能够处理简单任务并给出可读的完成说明；能够在 Agent 与 SOP 的帮助下持续推进，而不是每一步都来问你。

## 七、需要配套的支持材料

为了让这条训练路线真正生效，你还需要为新人准备一套配套材料，包括工具安装 SOP、账号申请 SOP、项目命名规则、Git 提交流程、服务器连接说明、数据库基础说明、部署平台说明，以及常见问题 FAQ。`onboarding_agent` 应基于这些材料生成每日引导，而不是临时编造答案。
